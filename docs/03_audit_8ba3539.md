### `_build_definitions()` 逻辑审查

ECDICT 的 `definition`（英文）和 `translation`（中文）都是多行文本（`\n` 分隔），需要按词性配对。

**匹配策略**（3轮）：
1. 按词性精确/前缀/别名匹配 en↔zh
2. 为缺 zh 的条目补充同词性已匹配的 translation
3. 剩余未匹配的 translation 追加为 en="" 的条目

**边界 case 测试**:
- both empty → `[]` ✅
- only translation → `[{pos, en:"", zh}]` ✅
- multi same-pos → 1:1 配对 ✅
- no pos prefix → 无法匹配，en/zh 分离 ⚠️（ECDICT 数据中大部分词条都有词性前缀，影响极小）

## 本次新发现问题

### P1-1: `api/data.py` 缺少 WAL 模式和线程锁

```python
_db_conn = sqlite3.connect(str(DICT_DB), check_same_thread=False)
```

FastAPI + uvicorn 在同步路由中会用线程池执行。全局单例 SQLite 连接无锁保护。虽然只读查询并发风险低，但在 WAL 模式下才能完全避免 `SQLITE_BUSY`。

**建议**:
```python
_db_conn = sqlite3.connect(str(DICT_DB), check_same_thread=False)
_db_conn.execute("PRAGMA journal_mode=WAL")
_db_conn.execute("PRAGMA readonly=1")
```

### P2-1: `ecdict.db` 部署无自动化

- DB 文件在 `.gitignore` 中（正确，300MB 不该入 git）
- 迁移文档说明了来源，但无自动下载/验证脚本
- 新部署者需手动下载 ECDICT CSV → 转换为 SQLite → 放到 `dictionary/` 目录
- **建议**: 添加 `scripts/setup-dictionary.sh` 或在 README 中补充一次性初始化步骤
